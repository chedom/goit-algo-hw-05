# Порівняння алгоритмів пошуку: Боєра-Мура, Кнута-Морріса-Пратта та Рабіна-Карпа за часом виконання
---
## Ціль експерименту порівняти три види алгоритмів сортування:
 - **Боєра-Мура** використовує евристики "поганого символу" ("хороший суфікс" не імплементовано) для пропуску секцій тексту
 - **Кнута-Морріса-Пратта** використовує префікс-функцію для уникнення повторних порівнянь вже збіжних символів
 - **Рабіна-Карпа** використовує поліноміальне хешування для швидкого порівняння підрядків

### Вхідні дані:
 - текст 1: технічний текст, містить фрагменти коду Java (з великою к-стю спеціальних символів, синтаксичних конструкцій та латиниці)
 - текст 2: текст з природною мовою з великим обсягом слів та пунктуацією

### Патерни для пошуку

| Текст   | Існуючий патерн    | Неіснуючий патерн | довгий існуючий патерн                   |
|---------|--------------------|-------------------|------------------------------------------|
| Текст 1 | `"Java"`           | `"private"`       |  -                                       |
| Текст 2 | `"розташовується"` | `"конвертація"`   | `"максимальна кількість вподобань 2048"` |


## Методика тестування

- Використано модуль `timeit` для точного вимірювання часу виконання
- Кожен алгоритм запускався **100 разів** для кожного патерну
- Результат — сумарний час виконання 100 ітерацій (у секундах)

## Аналіз результатів

| Алгоритм     | Текст 1 (Технічний/Код) | Текст 2 (Природна мова) |
|--------------|-------------------------|-------------------------|
| Боєра-Мура   | Найшвидший              | Найшвидший              |

### Алгоритм Боєра-Мура

- **Найшвидший** алгоритм в усіх тестових випадках
- Особливо ефективний для довших патернів та великих алфавітів
- Евристика "поганого символу" дозволяє пропускати великі ділянки тексту
- У найкращому випадку працює за `O(n/m)`, де `n` — довжина тексту, `m` — довжина патерну
- На довгих патернах (existing long) перевага над КМП зростає до **6.5x**
- Для коротких патернів перевага над КМП менш виражена

### Алгоритм Кнута-Морріса-Пратта (КМП)

- Стабільна продуктивність: гарантована складність `O(n + m)`
- Не залежить від розміру алфавіту
- Повільніший за Боєра-Мура через послідовне порівняння символів
- На довгих патернах продуктивність знижується відносно Боєра-Мура

### Алгоритм Рабіна-Карпа

- Простий у реалізації
- **Найповільніший** у всіх тестах через обчислення хешів
- Можливі хеш-колізії потребують додаткової перевірки
- Середня складність `O(n + m)`, але константний фактор вищий через операції хешування

## Загальний висновок
За результатами експерименту для обох типів текстів (як структурованого коду, так і неструктурованого природного тексту) найшвидшим алгоритмом визначено алгоритм Боєра-Мура. Він демонструє швидкість  меншу за O(N) на практиці, що робить його найкращим вибором для задач пошуку підрядка у великих текстових масивах загального призначення.
